40 /* Lock-free FastFlow allocator.
41  *
42  * Here we defined the ff_allocator (1) and the FFAllocator (2).
43  *
44  * 1. The ff_allocator allocates only large chunks of memory, slicing them up
45  * into little chunks all with the same size. Only one thread can perform
46  * malloc operations while any number of threads may perform frees using
47  * the ff_allocator.
48  *
49  * The ff_allocator is based on the idea of Slab Allocator, for more details
50  * about Slab Allocator please see:
51  * Bonwick, Jeff. "The Slab Allocator: An Object-Caching Kernel Memory
52  * Allocator." Boston USENIX Proceedings, 1994.
53  *
54  * 2. Based on the ff_allocator, the FFAllocator has been implemented.
55  * It might be used by any number of threads to dynamically
56  * allocate/deallocate memory. You have to include allocator.hpp and just use
57  * ff_malloc, ff_free, ff_realloc, ff_posix_memalign.
58  *
59  * Note: In all the cases it is possible, it is better to use the ff_allocator
60  * as it allows more control and is more efficient.
61  *
62  */

710  int entry = searchfb(key); // search for threadID in leak queue
